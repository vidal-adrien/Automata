About Automata

The Game of Life, also known simply as Life, is a cellular automaton devised by the British mathematician John Horton Conway in 1970. The "game" is a zero-player game, meaning that its evolution is determined by its initial state, requiring no further input. One interacts with the Game of Life by creating an initial configuration and observing how it evolves.

The game obeys two simple rules from which complex patterns emerge: An empty (dead) cell becomes alive if neighboured by 3 live cells and maintains this state while it is neighboured by 2 or 3 live cells. Although the first to be devised, this set of rules is but one of 218 possible rulesets called life-like cellular automata.

The rulesets are encoded by their B states (number of neighbours at which a cell transitions from death to life) and S states (number of neighbours at which a live cell remains alive).
Conway's game of life is thus noted B3/S23.

How To:

  • The central grid can be drawn on. The left click will draw live cells while the right click will erase them.

  • The grid can be navigated using combinations of Ctrl, Shift and the mouse wheel or the keyboard using the directional arrows and +/- keys.

  • will lauch the simulation. It will continue until the game reaches a stalemate of is paused by clicking again.

  • The speed of the simulation can be modified by tweaking the time interval (ms) between generations.
  • The dimensions of the grid can be changed as well as the colour of the cells. Beware that redimentioning will clear the grid.

  • The green counter at the top-right displays the number of generations and is reset whenever the grid is cleared or loaded with a pattern.
  • To open a patern and to save the current pattern. The rule used is saved with the grid layout. Aditionally, the pattern files can be loaded directly from the tree view. The root folder of the tree view can ca changed using . The change will be saved for future sessions.

  • Some example patterns are provided.
  • The ruleset can be edited, from the panel below the tree view, at any time but doing so will pause the simulation. The game can be set to only consider the cardinal 4 neighbours or all 8 neighbours. The rule editing form will only accept entries accordingly.
  • Some preset rulesets are available in a rolling list. Next to it, lets you save a ruleset to the list for future use while deletes the curently selected entry permanently. Adding an exixting ruleset will overwrite its name.

  • The edges of the grid can be "connected", meaning that the neighbouhood status is transmited from one edge to its oposite. This will have the effect of making patterns leaving one side of the board to go to the other. Another way to see this is to imagine the plane as the surface of a torus (see above).
  • The game always starts with cells colored black, a 50x50 grid and on the Conway (B3/S23) rule with 8 neighbours and unconnected edges as well as a 50ms-1 speed . If you would prefer other presets you can edit the "ruleset.ini" file. Note that a saved game keeps the settings as well as the pattern
